## 8. Limiti noti / known issues / risk

### Compile-time limits

- **`Tool` protocol require iOS 26+** — i 15 `FM*Tool` struct sono `@available(iOS 26.0, *)`. Su iPhone con iOS <26 il `respondWithTools` non gira; `GigiFallbackRouter` keyword-based subentra automatico (cablato in `GigiRequestRouter.route`).
- **`@Generable` no zero-arg init** — risolto: usiamo `Self.emptySlots()` esplicito con tutti i field = "" invece di `ActionSlots()`.
- **Compatibility shim `#if !canImport(FoundationModels)`** in `GigiRequestRouter.swift` per i pochi build target che non hanno FoundationModels. Deployment target iOS 26.2 fa sì che canImport sia sempre true in pratica.

### Runtime limits

- **Apple FM Tool calling NON è cablato** in `GigiRequestRouter.dispatchNativeTool` — oggi va via `GigiActionBridge.execute` direct con slots dal router decision (~80ms). Per attivare Apple FM Tool calling vero (`session.respondWithTools(tools: [matchingTool])`), aggiorna `dispatchNativeTool` per chiamare `GigiFoundationSession.shared.respondWithTools(text, tools, history)`. Saltato per stasera per evitare regression — l'effetto utente è identico, ma il path è meno "Apple FM-pure". Decide quando vuoi.
- **MCP harness-browser non wirato** in `ios-claude-agent.js` — manca lo spawn subprocess Claude Code via `claude-runner.js` con `options.mcpServers: ['harness-browser']`. Fallback su `GigiClaudeBridge.run` (legacy) garantisce continuità.
- **iOS Tool protocol API surface assumption** — ho assunto `LanguageModelSession(tools: [any Tool], instructions: String)` con `respond(to: String)` ritornante `.content`. Se la beta SDK iOS 26.3 differisce, `respondWithTools` può non compilare quando uno lo invoca (non chiamato ora, scaffold inerte). Build SUCCEEDED conferma compile-time OK.

### Memory budget Apple FM router

- Router system prompt ~3500 chars (~875 token)
- History sliding window (ultimo 6 turni) ~500 token
- User prompt ~50-200 token
- Output `FoundationRouterDecision` ~50 token
- **Totale stimato**: ~1500-2000 token su budget 4096 → OK con buffer

### Cost-aware routing fairness

- La regola `complexity <= 40 && !browser/code/vision` → `delegate_local` dipende dalla qualità di `complexityEstimate` di Apple FM. Se drifta (vedi piano §7 Q11 risk), aggiungeremo telemetria locale per ricalibrare.
